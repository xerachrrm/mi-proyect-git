# Etapas del Desarrollo de Software

Este documento describe las principales **etapas del ciclo de vida del desarrollo de software (SDLC)**.

## 1. Análisis de Requisitos

Es la fase inicial donde se recopila, analiza y documenta lo que el software debe hacer. Se define el alcance del proyecto y las necesidades de los usuarios.

### Requisitos Funcionales
Describen **qué debe hacer** el sistema (funcionalidades específicas).

### Requisitos No Funcionales
Describen **cómo debe comportarse** el sistema (rendimiento, seguridad, usabilidad, escalabilidad, etc.).

![Análisis de Requisitos](https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcTJGowPHsl8klQN72h-kobgqKivr3aJD9Dh8A&s)

---

## 2. Diseño

En esta etapa se define la arquitectura del sistema, los componentes, las interfaces de usuario, la base de datos y las tecnologías a utilizar. Se crean diagramas (UML, ER, etc.).

![Fase de Diseño](https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcRnRB4k3bEN-o31JR-O9IRvmCZCWPHk7o-UxA&s)

---

## 3. Codificación y Compilación

Se escribe el código fuente según las especificaciones del diseño. Posteriormente se compila y genera el ejecutable.

![Codificación](https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcR4SnUud4KcO_SgRpnX2O327sn6iPzeXdUyxA&s)

---

## 4. Pruebas

Se verifica que el software funcione correctamente y cumpla con los requisitos.

### Pruebas Unitarias
Prueban componentes individuales (funciones o clases) de forma aislada.

### Pruebas de Integración
Verifican que los diferentes módulos funcionen correctamente cuando se integran.

![Pruebas de Software](https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcTUWPmk7NpK0TijXVX-xLBRtJu3k_LZDpfZIg&s)

---

## 5. Explotación (Despliegue / Puesta en Producción)

Se instala el software en el entorno real para que los usuarios finales puedan utilizarlo.

![Despliegue](https://www.esic.edu/sites/default/files/2023-07/FotoJet%20%2829%29%20%281%29.jpg)

---

## 6. Mantenimiento

Fase posterior al despliegue. Incluye corrección de errores, mejoras, actualizaciones y adaptación a nuevos requisitos.

![Mantenimiento](https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcS1E_3ItCWPiL8NdGnGwfA6KPYnKiGkRtV0aQ&s)

---

## 7. Documentación - Tipos de Documentos

La documentación es esencial a lo largo de todo el ciclo de vida.

**Principales tipos de documentos:**
- Documento de Requisitos (SRS)
- Documento de Diseño (SDD)
- Manual de Usuario
- Manual Técnico / de Administración
- Plan de Pruebas
- Reportes de Incidencias
- Documentación del Código (comentarios, README)

![Documentación](https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcTQwumb7QFKu6sTHBFQ4bctuw8Hg-61dsSzbg&s)
